SensorRUNv02 Post Analysis

Script for running post-analysis functions (e.g. plotting, reporting, additional analyses) on output from SensorRUNv02.

SensorRUNv02 is a MATLAB pipeline for analyzing biomechanics during running using inertial sensors. It is built on the RemoteBMX pipeline for biomechanical analysis. See the SensorRUNv02 README for more information.

Installation

Download “sensorRUNv02_postanalysis” directory and add to the MATLAB path.

Requirements

  • MATLAB version R2018a or later
  • MATLAB Signal Processing toolbox
  • Post-analysis functions located in the “postanalysis” folder, which should be in the same folder as the RemoteBMX folder.

Input

.mat file containing the results of the SensorRUNv02 pipeline, which is a struct called “session” or the “session” struct already loaded into the Matlab workspace

Output

Depended the post-analysis function. See each script for details

What It Does

Loads the “session” struct, if needed.

Creates a list of available post-analysis functions for user selection. Note: all post-analysis functions must be in the “postanalysis folder” that is located in the same director as the “RemoteBMX” folder.

Runs the post-analysis function selected by the user. When completed, it prompts the user if they want to conduct additional analyses. If yes, it re-presents the post-analysis function list to the user. If not, it quits.

How to Run

  1. In the MATLAB command window, type ‘sensorRUNv02_postanalysis’ and press ‘enter’.

  2. Follow the prompts to load a .mat file containing the SensorRUNv02 “session” struct or indicate that the “session” struct has already been loaded into the Matlab workspace.

  3. Select the desired post-analysis function from the dropdown list.

  4. Follow the steps required for the post-analysis function.

  5. When the post-analysis function has completed, respond to the prompt to indicate if additional analyses are desired and if yes, follow steps 3 & 4 to select the new analysis function. Note that the only way to select a new dataset for post-analysis is to select “No” for “Additional Analyses?” and then call the post analysis script again, starting with step 1.
