SensorRUNv02 Post Analysis

Script for visualizing stride-segmented sensor data output from SensorRUNv02.

SensorRUNv02 is a MATLAB pipeline for analyzing biomechanics during running using inertial sensors. It is built on the RemoteBMX pipeline for biomechanical analysis. See the SensorRUNv02 README for more information.

Installation

Download and add to the postanalysis folder.

Requirements

  • MATLAB version R2018a or later
  • MATLAB Signal Processing toolbox

Input

.mat file containing the results of the SensorRUNv02 pipeline, which is a struct called “session” or the “session” struct already loaded into the Matlab workspace.

Output

Figures of data by stride for selected sensor locations x channels x running bouts. The ensemble mean and sd deviation across strides are shown on each graph. X-axis: percent of stride from foot contact to the instant before next ipsilateral foot contact. Y-axis: selected sensor data

What It Does

Allows the user to select the running bouts, sensor locations, data channels, and optional low-pass filtering to apply to the data.

Plots each selected data by stride (foot contact to next foot contact) along with ensemble mean and standard deviation.

How to Run

  1. In the MATLAB command window, type ‘sensorRUNv02_postanalysis’ and press ‘enter’.

  2. Follow the prompts to load a .mat file containing the SensorRUNv02 “session” struct or indicate that the “session” struct has already been loaded into the Matlab workspace.

  3. Select sensorRUNv02_visualizeIMUData from the dropdown list.

  4. Select the running bouts to plot. Multiple bouts can be selected using CTRL/CMD.

  5. Select the sensor locations to plot. Multiple locations can be selected using CTRL/CMD.

  6. For each sensor location, select the data to plot. Multiple datatypes can be selected using CTRL/CMD.

    The list is populated with all datatypes in session.run.data.imu.locations.(locationName) and the resultant vector magnitude for each data types, denoted by “mag”.

  7. Select yes to low-pass filter the data. All data selected will be filtered. The filter applied is a fourth-order, zero-lag Butterworth low-pass filter.

    If Yes is selected, a prompt will appear to enter the cut-off frequency in Hz.

  8. A dropdown will appear to select the filename containing the IMU foot contact data. Select the appropriate field.

  9. Figures will appear for each run x location x data.

    Gray lines are individual strides. The thick red line is the ensemble mean, and the shaded red region with thinner lines is +/- 1 standard deviation. Data are normalized to 0–100% of stride (ipsilateral foot contact to next ipsilateral foot contact).

    When the post-analysis function has completed, respond to the prompt to indicate if additional analyses are desired. If yes, follow steps 3 & 4 to select the new analysis function. Note that the only way to select a new dataset for post-analysis is to select “No” for “Additional Analyses?” and then call the post analysis script again, starting with step 1.
